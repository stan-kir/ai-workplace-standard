# 14. Применение стандарта в оценке сотрудников

> Рабочие материалы: [шаблон оценки сотрудника (Приложение 9)](../20-appendices/20-appendices.md#209-приложение-9-шаблон-оценки-сотрудника), [шаблон self-review (Приложение 10)](../20-appendices/20-appendices.md#2010-приложение-10-шаблон-self-review-сотрудника), [шкала зрелости A1–A6](../05-maturity-scale/05-maturity-scale.md), [red flags](../17-red-flags/17-red-flags.md).

Этот раздел фиксирует, как оценивать AI/agent-first зрелость сотрудников не по декларациям, а по поведению, результатам и изменению способа работы.

Главная идея:

> Оценивается не факт использования ИИ, а способность сотрудника быстрее, качественнее и самостоятельнее достигать результата через ИИ, агентов, шаблоны и автоматизацию.

---

## 14.1. Зачем стандарт нужен в оценке

Стандарт помогает отличить реальную зрелость от имитации.

| Что не оцениваем как зрелость | Что оцениваем                                 |
| ----------------------------- | --------------------------------------------- |
| «Пользуется ChatGPT»          | Решает рабочие задачи через ИИ                |
| «Знает названия инструментов» | Применяет инструменты к реальным процессам    |
| «Прошёл обучение»             | Изменил способ выполнения задач               |
| «Сделал красивый текст»       | Получил применимый результат                  |
| «Собрал сложный сценарий»     | Дал измеримый эффект без лишней сложности     |
| «Говорит про агентов»         | Снял рутину, ускорил работу, улучшил качество |

---

## 14.2. Что именно оценивается

Оценка строится по 8 блокам.

| Блок                        | Что проверяем                                                      |
| --------------------------- | ------------------------------------------------------------------ |
| **1. Применение**           | Использует ли сотрудник ИИ в реальных рабочих задачах              |
| **2. Постановка задачи**    | Умеет ли дать цель, контекст, данные, ограничения, формат          |
| **3. Работа с материалами** | Умеет ли работать с документами, таблицами, данными, источниками   |
| **4. Проверка результата**  | Проверяет ли факты, логику, цифры, риски                           |
| **5. Безопасность**         | Соблюдает ли правила данных и конфиденциальности                   |
| **6. Повторяемость**        | Создаёт ли шаблоны, сценарии, инструкции, чек-листы                |
| **7. Эффект**               | Снижает ли ручную работу, ускоряет ли задачи, повышает ли качество |
| **8. Ответственность**      | Не перекладывает ли ошибки и решения на ИИ                         |

---

## 14.3. Шкала оценки по каждому блоку

Для простоты можно использовать шкалу 1–5.

| Оценка | Значение                                                    |
| -----: | ----------------------------------------------------------- |
|  **1** | Не применяет / сопротивляется / не понимает                 |
|  **2** | Применяет эпизодически, слабо и только по подсказке         |
|  **3** | Применяет регулярно на базовом уровне                       |
|  **4** | Применяет уверенно, создаёт повторяемые сценарии            |
|  **5** | Меняет способ работы вокруг себя, влияет на команду/процесс |

---

## 14.4. Матрица оценки сотрудника

| Критерий                 | 1                      | 3                               | 5                                              |
| ------------------------ | ---------------------- | ------------------------------- | ---------------------------------------------- |
| **Применение ИИ**        | Не использует          | Использует в регулярных задачах | Встраивает в способ работы команды/процесса    |
| **Постановка задачи**    | Задаёт общие вопросы   | Даёт цель, контекст, формат     | Проектирует полноценные сценарии               |
| **Работа с материалами** | Не умеет               | Работает с текстами/таблицами   | Строит повторяемую обработку данных/документов |
| **Проверка результата**  | Копирует без проверки  | Проверяет базовые ошибки        | Создаёт систему контроля качества              |
| **Безопасность**         | Не понимает риски      | Соблюдает правила               | Учитывает риски при проектировании сценариев   |
| **Повторяемость**        | Разовые запросы        | Использует шаблоны              | Создаёт AI-сценарии для других                 |
| **Эффект**               | Нет измеримого эффекта | Экономит время в своих задачах  | Снимает рутину с команды/процесса              |
| **Ответственность**      | «ИИ ошибся»            | Проверяет и дорабатывает        | Управляет результатом и ограничениями          |

---

## 14.5. Оценка по уровням зрелости A1–A6

| Уровень | Как распознать                                                        |
| ------- | --------------------------------------------------------------------- |
| **A1**  | Использует ИИ для простых текстов, идей, объяснений                   |
| **A2**  | Умеет ставить рабочие задачи, давать контекст, проверять результат    |
| **A3**  | Создаёт шаблоны, промпты, повторяемые AI-сценарии                     |
| **A4**  | Управляет AI-сценариями в команде, контролирует качество              |
| **A5**  | Перестраивает процесс, считает эффект, снижает ручную работу системно |
| **A6**  | Меняет модель работы функции/направления через agent-first подход     |

---

## 14.6. Минимальные уровни по ролям

| Уровень роли        | Минимум | Что должно быть видно в оценке                       |
| ------------------- | ------: | ---------------------------------------------------- |
| **Специалист**      |      A2 | Использует ИИ в рабочих задачах, проверяет результат |
| **Ведущий эксперт** |      A3 | Создаёт экспертные шаблоны, чек-листы, сценарии      |
| **Старший / лид**   |      A3 | Помогает команде, создаёт повторяемые сценарии       |
| **Руководитель**    |      A4 | Внедряет AI-сценарии в работу команды                |
| **ТОП**             |      A5 | Перестраивает процессы функции через ИИ и агентов    |

---

## 14.7. Что сотрудник должен показать на оценке

Не общие слова, а доказательства.

| Доказательство      | Пример                                         |
| ------------------- | ---------------------------------------------- |
| **Задачи**          | Какие рабочие задачи выполнял через ИИ         |
| **Запросы**         | Как формулировал задачи и контекст             |
| **Материалы**       | Какие документы, таблицы, данные обрабатывал   |
| **Проверка**        | Как проверял результат                         |
| **Сценарии**        | Какие шаблоны или AI-сценарии создал           |
| **Эффект**          | Сколько времени, ошибок или ручной работы снял |
| **Ограничения**     | Где ИИ не справился и что было сделано         |
| **Передача другим** | Кто ещё использует созданный сценарий          |

---

## 14.8. Примеры хороших доказательств

## Специалист

> Использовал корпоративный AI-чат для подготовки еженедельной сводки. Сократил время с 2 часов до 50 минут. Проверяет результат по чек-листу. Конфиденциальные данные не загружает во внешние сервисы.

## Ведущий эксперт

> Создал чек-лист AI-проверки договоров/материалов/отчётов. Команда использует его перед передачей результата руководителю. Ошибки стали выявляться раньше.

## Старший / лид

> Собрал шаблон анализа типовой проблемы и инструкцию для команды. Три сотрудника используют сценарий регулярно. Время подготовки материалов сократилось.

## Руководитель

> Составил карту ручной рутины отдела, выбрал 3 сценария, один внедрил. Еженедельная сводка теперь готовится через агента, руководитель проверяет выводы и принимает решения.

## ТОП

> Определил 5 процессов функции для agent-first пересборки, запустил 2 пилота, зафиксировал эффект, назначил владельцев, включил AI-зрелость в требования к руководителям.

---

## 14.9. Примеры плохих доказательств

| Формулировка                       | Почему слабо                  |
| ---------------------------------- | ----------------------------- |
| «Я пользуюсь ChatGPT»              | Нет рабочего результата       |
| «ИИ помогает мне думать»           | Нет конкретики                |
| «Я иногда прошу написать текст»    | Бытовой уровень               |
| «Мы обсуждали автоматизацию»       | Нет внедрения                 |
| «Я прошёл курс»                    | Обучение не равно применению  |
| «Сделали красивую презентацию»     | Неясен эффект                 |
| «ИИ ошибся, поэтому не используем» | Нет зрелого контроля качества |
| «Ждём ИТ»                          | Нет MVP и владельца процесса  |

---

## 14.10. Оценка поведения

AI/agent-first зрелость — это не только инструменты, но и поведение.

| Поведение                                       | Оценка          |
| ----------------------------------------------- | --------------- |
| Сначала пробует решить задачу через ИИ/сценарий | Плюс            |
| Показывает, что пробовал и где ограничение      | Плюс            |
| Создаёт шаблоны для повторяемых задач           | Плюс            |
| Делится сценариями с командой                   | Плюс            |
| Проверяет результат и признаёт ограничения      | Плюс            |
| Не использует ИИ без напоминания                | Минус           |
| Делает руками по привычке                       | Минус           |
| Копирует ответ без проверки                     | Минус           |
| Прячется за «я не умею»                         | Минус           |
| Перекладывает ошибку на ИИ                      | Минус           |
| Загружает данные без правил                     | Критичный минус |

---

## 14.11. Оценка руководителя

Руководитель оценивается не только по личному применению ИИ.

## Что проверять

| Критерий            | Вопрос                                           |
| ------------------- | ------------------------------------------------ |
| **Карта рутины**    | Видит ли ручную работу команды?                  |
| **Сценарии**        | Какие AI-сценарии запущены?                      |
| **Команда**         | Используют ли сотрудники ИИ регулярно?           |
| **Качество**        | Есть ли проверка AI-результатов?                 |
| **Эффект**          | Что улучшилось в скорости, качестве, нагрузке?   |
| **Обучение**        | Обучает ли сотрудников через реальные задачи?    |
| **Сопротивление**   | Как работает с теми, кто не применяет стандарт?  |
| **Масштабирование** | Перешли ли от разового использования к процессу? |

## Плохая оценка руководителя

Руководитель:

* сам пользуется ИИ, но команда нет;
* не знает, какие задачи можно снять с людей;
* не имеет карты ручной рутины;
* ждёт ИТ;
* не измеряет эффект;
* не требует применения;
* не проверяет качество.

## Хорошая оценка руководителя

Руководитель:

* внедрил AI-сценарии в регулярную работу;
* снял часть ручной рутины;
* обучил команду;
* создал шаблоны;
* контролирует качество;
* измеряет эффект;
* улучшает процесс.

---

## 14.12. Оценка ТОПа

ТОП оценивается не по тому, использует ли он ИИ лично.

## Что проверять

| Критерий               | Вопрос                                             |
| ---------------------- | -------------------------------------------------- |
| **Видение функции**    | Понимает ли, как ИИ меняет функцию?                |
| **Процессы**           | Какие процессы выбраны для agent-first пересборки? |
| **Экономика**          | Есть ли оценка эффекта?                            |
| **Роли**               | Меняются ли требования к людям?                    |
| **Управление**         | Требует ли AI-сценарии от руководителей?           |
| **Портфель инициатив** | Есть ли набор пилотов и приоритетов?               |
| **Governance**         | Учитываются ли безопасность, данные, контроль?     |
| **Масштабирование**    | Есть ли переход от экспериментов к системе?        |

Плохой признак:

> ТОП говорит про ИИ как про инструмент личной эффективности, но не меняет процессы, роли и требования к руководителям.

Хороший признак:

> ТОП меняет операционную модель функции: что делают люди, что делают агенты, что автоматизируется, какие метрики управляют системой.

---

## 14.13. Встроить в performance review

В performance review можно добавить отдельный блок:

## AI/agent-first вклад

1. Какие задачи сотрудник ускорил через ИИ?
2. Какие ручные операции снял?
3. Какие шаблоны, сценарии или агенты создал?
4. Кто ещё использует созданные решения?
5. Как проверял качество результата?
6. Какие риски учёл?
7. Какой измеримый эффект получил?
8. Какой следующий сценарий предлагает?

---

## 14.14. Итоговая оценочная форма

| Критерий                        |      Оценка 1–5 | Комментарий | Доказательство |
| ------------------------------- | --------------: | ----------- | -------------- |
| Использует ИИ в рабочих задачах |                 |             |                |
| Умеет ставить задачи ИИ/агенту  |                 |             |                |
| Работает с документами/данными  |                 |             |                |
| Проверяет результат             |                 |             |                |
| Соблюдает безопасность          |                 |             |                |
| Создаёт шаблоны/сценарии        |                 |             |                |
| Снимает ручную рутину           |                 |             |                |
| Делится решениями с командой    |                 |             |                |
| Измеряет эффект                 |                 |             |                |
| Соответствует уровню роли       | Да / Риск / Нет |             |                |

---

## 14.15. Интерпретация итоговой оценки

| Итог                       | Значение                                      | Действие                                 |
| -------------------------- | --------------------------------------------- | ---------------------------------------- |
| **Высокое соответствие**   | Сотрудник выше стандарта роли                 | Давать сложнее задачи, включать в пилоты |
| **Соответствие**           | Сотрудник выполняет требования роли           | Поддерживать развитие                    |
| **Частичное соответствие** | Есть пробелы, но есть прогресс                | План развития на 30–60 дней              |
| **Риск**                   | Сотрудник не дотягивает и слабо прогрессирует | Управленческая обратная связь            |
| **Несоответствие**         | Сопротивление или отсутствие применения       | Рассматривать как риск соответствия роли |

---

## 14.16. Что делать при низкой оценке

| Причина                            | Действие                                 |
| ---------------------------------- | ---------------------------------------- |
| Не знает инструментов              | Обучение и практика                      |
| Не умеет ставить задачу            | Дать шаблон tasking и разобрать примеры  |
| Не проверяет результат             | Ввести обязательный чек-лист             |
| Боится ИИ                          | Начать с безопасных простых задач        |
| Делает руками по привычке          | Показать сценарии и требовать применения |
| Сопротивляется                     | Управленческая обратная связь            |
| Имитирует применение               | Проверка на реальных задачах             |
| Нарушает безопасность              | Ограничить доступ, провести разбор       |
| Руководитель не внедряет в команду | Оценивать как управленческий риск        |

---

## 14.17. Главный вывод раздела

> Оценка AI/agent-first зрелости должна показывать не то, насколько сотрудник «в теме ИИ», а насколько он реально меняет способ выполнения работы: меньше ручной рутины, выше скорость, лучше качество, больше самостоятельности и больше повторяемых сценариев.
