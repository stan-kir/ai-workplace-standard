# 13. Применение стандарта в обучении и развитии сотрудников

Этот раздел фиксирует, как обучать сотрудников работе в AI/agent-first среде.

Главная идея:

> Обучение ИИ должно быть не «курс про ChatGPT», а система развития новой производительности: сотрудник учится снимать рутину, ставить задачи агентам, проверять результат и создавать повторяемые сценарии.

---

## 13.1. Зачем стандарт нужен в обучении

Стандарт нужен, чтобы обучение не превратилось в набор разрозненных мастер-классов по инструментам.

Обучение должно давать:

| Цель                        | Что это означает                                            |
| --------------------------- | ----------------------------------------------------------- |
| **Единый язык**             | Все одинаково понимают ИИ, агентов, сценарии, автоматизацию |
| **Минимальный уровень**     | Каждый сотрудник выходит на обязательный AI-минимум роли    |
| **Практику**                | Обучение строится на реальных задачах, а не на теории       |
| **Снятие рутины**           | Главный результат — меньше ручной работы                    |
| **Безопасность**            | Сотрудники понимают, что можно и нельзя делать              |
| **Управленческую зрелость** | Руководители учатся внедрять ИИ в работу команды            |
| **Повторяемость**           | Появляются шаблоны, сценарии, агенты, инструкции            |

---

## 13.2. Принцип обучения

Неправильная модель:

> Послушали лекцию → узнали про инструменты → вернулись к старой работе.

Правильная модель:

> Взяли реальную задачу → применили ИИ → проверили результат → сделали шаблон → внедрили в работу → измерили эффект.

---

## 13.3. Структура обучения

Обучение должно состоять из 5 блоков.

| Блок                       | Для кого                                | Цель                                       |
| -------------------------- | --------------------------------------- | ------------------------------------------ |
| **Базовый курс**           | Все сотрудники                          | Выйти на A1–A2                             |
| **Практика по ролям**      | Все роли                                | Применить ИИ в своей работе                |
| **Трек для руководителей** | Руководители, лиды                      | Внедрять AI-сценарии в команде             |
| **Трек для экспертов**     | Ведущие, аналитики, профильные эксперты | Упаковывать экспертизу в сценарии          |
| **Продвинутый трек**       | A3+ сотрудники                          | Создавать workflow, агентов, автоматизации |

---

## 13.4. Базовый курс для всех сотрудников

## Цель

Вывести каждого сотрудника на минимальный уровень A2.

## Что входит

1. Что такое AI/agent-first среда.
2. Чем ИИ отличается от цифрового агента.
3. Как ставить задачу ИИ.
4. Как давать контекст.
5. Как работать с документами и таблицами.
6. Как проверять результат.
7. Что нельзя загружать в ИИ.
8. Как использовать готовые сценарии.
9. Как не перекладывать ответственность на ИИ.
10. Как применять ИИ в ежедневной работе.

## Практический результат

После базового курса сотрудник должен уметь:

* сформулировать рабочий запрос;
* использовать корпоративный AI-инструмент;
* получить черновик результата;
* проверить результат;
* доработать итог;
* объяснить, что сделал и где риски.

---

## 13.5. Практика по ролям

После базового курса обучение должно переходить в ролевую практику.

| Роль                | Практическая задача                                    |
| ------------------- | ------------------------------------------------------ |
| **Специалист**      | Выполнить типовую задачу через ИИ по шаблону           |
| **Ведущий эксперт** | Создать экспертный чек-лист или шаблон проверки        |
| **Старший / лид**   | Создать повторяемый AI-сценарий для группы             |
| **Руководитель**    | Найти рутину команды и запустить первый AI-сценарий    |
| **ТОП**             | Определить процессы функции для agent-first пересборки |

---

## 13.6. Трек для руководителей

## Цель

Научить руководителей не просто пользоваться ИИ, а внедрять agent-first модель в работу команды.

## Темы

1. Как найти ручную рутину в отделе.
2. Как отличить задачу, сценарий, процесс и автоматизацию.
3. Как выбрать первый AI-сценарий.
4. Как запустить MVP без большого ИТ-проекта.
5. Как обучать команду через реальные задачи.
6. Как контролировать качество AI-результатов.
7. Как измерять эффект.
8. Как управлять сопротивлением.
9. Как не допустить хаоса и имитации.
10. Как встроить ИИ в регулярный управленческий ритм.

## Практический результат

Каждый руководитель должен подготовить:

* карту ручной рутины отдела;
* список 3–5 AI-сценариев;
* один сценарий для запуска;
* критерии качества;
* метрики эффекта;
* план внедрения на 30 дней.

---

## 13.7. Трек для ведущих экспертов

## Цель

Научить экспертов масштабировать свою экспертизу через ИИ, шаблоны, базы знаний и сценарии.

## Темы

1. Как использовать ИИ для экспертного анализа.
2. Как искать ошибки, риски и слабые места.
3. Как отделять факт, гипотезу и интерпретацию.
4. Как проверять AI-выводы в своей области.
5. Как создавать экспертные чек-листы.
6. Как превращать личный опыт в повторяемый сценарий.
7. Как создавать базу типовых ошибок.
8. Как обучать команду через экспертные шаблоны.

## Практический результат

Ведущий эксперт должен создать минимум один артефакт:

* экспертный чек-лист;
* шаблон анализа;
* AI-сценарий проверки;
* базу типовых ошибок;
* инструкцию для команды;
* промпт-пакет по своей области.

---

## 13.8. Трек для старших / лидов

## Цель

Научить лидов создавать повторяемые рабочие сценарии и помогать команде применять ИИ.

## Темы

1. Как разложить задачу на шаги.
2. Как написать промпт-шаблон.
3. Как оформить инструкцию для команды.
4. Как проверить качество результата.
5. Как собрать простой AI-сценарий.
6. Как обучить младших сотрудников.
7. Как улучшать сценарий по обратной связи.
8. Как масштабировать успешный шаблон.

## Практический результат

Лид должен создать:

* один рабочий AI-сценарий;
* инструкцию к нему;
* чек-лист проверки;
* пример хорошего результата;
* пример плохого результата;
* рекомендации для команды.

---

## 13.9. Продвинутый трек A3+

## Цель

Подготовить сотрудников, которые смогут создавать более сложные AI-сценарии, workflow, агентов и прототипы.

## Темы

1. Workflow automation.
2. No-code / low-code инструменты.
3. Работа с API и вебхуками на базовом уровне.
4. Создание простых агентов.
5. Работа с RAG и базами знаний.
6. Подключение таблиц, документов, задач, форм.
7. Human-in-the-loop.
8. Логирование и контроль качества.
9. Ограничения безопасности.
10. Тестирование сценариев.

## Практический результат

Участник должен собрать MVP:

* простого workflow;
* агента;
* автоматизированной сводки;
* внутреннего помощника;
* сценария обработки данных;
* сценария проверки качества.

---

## 13.10. Форматы обучения

Обучение должно быть коротким, прикладным и регулярным.

| Формат                     | Когда использовать                  |
| -------------------------- | ----------------------------------- |
| **Вводный курс**           | Для всех сотрудников                |
| **Практикум**              | Для отработки на рабочих задачах    |
| **Разбор кейсов**          | Для руководителей и экспертов       |
| **Мастерская сценариев**   | Для создания AI-сценариев команд    |
| **Разбор ошибок**          | Для повышения качества              |
| **Внутренние демо**        | Чтобы показывать успешные сценарии  |
| **Библиотека примеров**    | Чтобы сотрудники не начинали с нуля |
| **AI-часы / консультации** | Для помощи командам на старте       |

---

## 13.11. Библиотека обучения

Компания должна создать и поддерживать библиотеку материалов.

Минимальный набор:

| Артефакт                         | Зачем нужен                             |
| -------------------------------- | --------------------------------------- |
| **Базовая инструкция по ИИ**     | Быстрый старт для всех                  |
| **Правила безопасности**         | Что можно и нельзя делать               |
| **Шаблон запроса к ИИ**          | Единый стандарт постановки задач        |
| **Чек-лист проверки результата** | Контроль качества                       |
| **Библиотека промптов**          | Быстрое применение                      |
| **Библиотека AI-сценариев**      | Повторяемая работа                      |
| **Примеры хороших результатов**  | Эталон качества                         |
| **Примеры ошибок**               | Обучение на рисках                      |
| **Инструкции по инструментам**   | Как пользоваться разрешёнными сервисами |
| **FAQ**                          | Ответы на частые вопросы                |

---

## 13.12. Обучение через реальные задачи

Главное правило:

> Каждый модуль обучения должен завершаться рабочим результатом, который можно использовать в компании.

Плохой результат обучения:

> Сотрудник прослушал курс и получил сертификат.

Хороший результат обучения:

> Сотрудник создал сценарий, шаблон, инструкцию, чек-лист или улучшил реальную задачу.

---

## 13.13. Минимальный учебный цикл

Каждое обучение строится по циклу:

1. Выбрать реальную задачу.
2. Разложить её на шаги.
3. Определить, где может помочь ИИ.
4. Сформулировать запрос.
5. Получить первый результат.
6. Проверить ошибки.
7. Доработать.
8. Оформить как повторяемый сценарий.
9. Протестировать на 3–5 примерах.
10. Передать в библиотеку команды.

---

## 13.14. Оценка результата обучения

Оценивать нужно не «понравилось / не понравилось», а изменение поведения.

| Критерий                | Что проверяется                               |
| ----------------------- | --------------------------------------------- |
| **Применение**          | Использует ли сотрудник ИИ в реальных задачах |
| **Качество постановки** | Умеет ли давать контекст и критерии           |
| **Проверка результата** | Не копирует ли ответ бездумно                 |
| **Безопасность**        | Не нарушает ли правила данных                 |
| **Повторяемость**       | Создаёт ли шаблоны и сценарии                 |
| **Эффект**              | Сократилось ли время, ошибки, ручная работа   |
| **Самостоятельность**   | Меньше ли зависимость от руководителя/ИТ      |
| **Передача другим**     | Может ли обучить коллег                       |

---

## 13.15. Минимальные уровни после обучения

| Категория сотрудников | Ожидаемый уровень после базового обучения |
| --------------------- | ----------------------------------------- |
| **Специалисты**       | A2                                        |
| **Ведущие эксперты**  | A3                                        |
| **Старшие / лиды**    | A3                                        |
| **Руководители**      | A4                                        |
| **ТОПы**              | A5                                        |

---

## 13.16. Как выявлять сотрудников с высоким потенциалом

Сотрудник с высоким AI-потенциалом:

* быстро пробует новые инструменты;
* сам находит рутину;
* создаёт шаблоны;
* делится сценариями;
* проверяет результат;
* не ждёт идеальной инструкции;
* задаёт правильные вопросы по данным и доступам;
* умеет объяснить другим;
* считает эффект;
* не делает из ИИ игрушку или магию.

Таких сотрудников стоит включать в:

* кадровый резерв;
* внутреннюю сеть AI-амбассадоров;
* пилоты;
* проектные группы;
* разработку корпоративных сценариев;
* обучение коллег.

---

## 13.17. AI-амбассадоры внутри компании

Для ускорения внедрения можно выделить AI-амбассадоров.

## Роль AI-амбассадора

AI-амбассадор — это сотрудник, который помогает своей функции или команде быстрее осваивать ИИ, находить сценарии, запускать первые эксперименты и передавать лучшие практики.

Он не заменяет ИТ, руководителя или владельца процесса.

## Что делает AI-амбассадор

* показывает примеры;
* помогает формулировать запросы;
* собирает библиотеку сценариев;
* помогает протестировать MVP;
* фиксирует ошибки;
* передаёт обратную связь владельцам AI-платформы;
* помогает команде перейти от «попробовали ИИ» к регулярному применению.

## Кого выбирать

Лучшие кандидаты:

* A3+;
* высокая обучаемость;
* практический склад мышления;
* авторитет в команде;
* отсутствие технофанатизма;
* способность объяснять простым языком;
* дисциплина в безопасности.

---

## 13.18. Что не должно происходить в обучении

| Ошибка                                | Почему опасно                                   |
| ------------------------------------- | ----------------------------------------------- |
| Учить только промптам                 | Сотрудники останутся на уровне «чат для текста» |
| Учить только инструментам             | Инструменты меняются, принцип работы важнее     |
| Делать длинные теоретические курсы    | Не меняет поведение                             |
| Не привязывать к задачам роли         | Не будет применения                             |
| Не проверять результат                | Будет имитация обучения                         |
| Не учитывать безопасность             | Риск утечек                                     |
| Не обучать руководителей отдельно     | Команды вернутся в старую модель                |
| Не создавать библиотеку сценариев     | Каждый будет изобретать заново                  |
| Не измерять эффект                    | Непонятно, работает ли обучение                 |
| Делать обучение добровольным для всех | Стандарт не станет нормой                       |

---

## 13.19. Роль руководителя в обучении

Руководитель обязан:

* направлять сотрудников на обучение;
* давать реальные задачи для практики;
* проверять применение после обучения;
* требовать использования сценариев;
* выделять ручную рутину для автоматизации;
* поощрять создание шаблонов;
* пресекать имитацию;
* фиксировать эффект.

Обучение без управленческого требования не меняет систему.

---

## 13.20. Главный вывод раздела

> Обучение AI/agent-first работе должно измеряться не количеством пройденных курсов, а количеством рутины, снятой с сотрудников, количеством внедрённых сценариев, ростом скорости, качества и самостоятельности команд.
