# 20. Приложения и рабочие шаблоны

Этот раздел содержит прикладные формы, которые позволяют использовать стандарт не как теоретический документ, а как рабочий инструмент для сотрудников, руководителей, HR, HRBP, ИТ и владельцев процессов.

Главная идея:

> Стандарт должен быстро превращаться в действие: запрос, сценарий, проверку, оценку, обучение, найм, адаптацию и метрику эффекта.

---

## 20.1. Приложение 1. Краткая шкала AI/agent-first зрелости

| Уровень | Название                 | Главный признак                                           |
| ------- | ------------------------ | --------------------------------------------------------- |
| **A1**  | AI User                  | Использует ИИ для простых личных задач                    |
| **A2**  | Agent Operator           | Умеет ставить рабочие задачи ИИ/агенту                    |
| **A3**  | Workflow Builder         | Создаёт повторяемые AI-сценарии и шаблоны                 |
| **A4**  | Agent Supervisor         | Управляет агентами, сценариями и качеством в команде      |
| **A5**  | Agentic Process Owner    | Перестраивает процессы вокруг ИИ, агентов и автоматизации |
| **A6**  | AI Transformation Leader | Меняет модель работы функции или компании                 |

## 20.3. Приложение 3. Базовый шаблон запроса к ИИ

## Формула

> **Роль → задача → контекст → данные → ограничения → формат → критерии качества.**

## Шаблон

```text
Ты выступаешь как [роль].

Твоя задача — [что нужно сделать].

Контекст:
[кратко описать ситуацию, цель, аудиторию, зачем нужен результат].

Входные данные:
[какие данные, документы, таблицы, вводные используются].

Ограничения:
[что важно учесть, чего нельзя делать, какие риски].

Формат результата:
[таблица / список / план / письмо / вывод / инструкция / презентационная структура].

Критерии качества:
[как понять, что результат хороший: краткость, точность, применимость, проверяемость, отсутствие воды].
```

## Пример

```text
Ты выступаешь как операционный аналитик.

Твоя задача — проанализировать выгрузку по задачам отдела за неделю.

Контекст:
Руководителю нужна краткая сводка для планёрки. Нужно быстро увидеть просрочки, зависшие задачи и риски.

Входные данные:
Таблица с задачами, ответственными, сроками, статусами и комментариями.

Ограничения:
Не делай выводы без данных. Отдельно помечай гипотезы. Не придумывай причины, если их нет в таблице.

Формат результата:
Таблица: проблема / данные / возможная причина / что проверить / предлагаемое действие.

Критерии качества:
Кратко, конкретно, без общих фраз, с фокусом на управленческие действия.
```

## 20.4. Приложение 4. Чек-лист проверки AI-результата

## Короткая формула

> **Подходит? Верно? Безопасно? Применимо? Достаточно?**

| Вопрос          | Проверка                                                     |
| --------------- | ------------------------------------------------------------ |
| **Подходит?**   | Результат отвечает на исходную задачу?                       |
| **Верно?**      | Нет ли ошибок, выдуманных фактов, неверных цифр?             |
| **Безопасно?**  | Не нарушены ли правила данных и конфиденциальности?          |
| **Применимо?**  | Можно ли использовать результат в реальной работе?           |
| **Достаточно?** | Не нужна ли доработка, согласование или экспертная проверка? |

## Расширенный чек-лист

| Блок               | Вопрос                                                                 |
| ------------------ | ---------------------------------------------------------------------- |
| Задача             | ИИ решил именно поставленную задачу?                                   |
| Контекст           | Учтены ли вводные и ограничения?                                       |
| Факты              | Проверены ли фактические утверждения?                                  |
| Цифры              | Проверены ли расчёты и исходные данные?                                |
| Логика             | Нет ли противоречий?                                                   |
| Формат             | Можно ли передавать результат дальше?                                  |
| Риски              | Нет ли юридических, финансовых, репутационных или операционных рисков? |
| Конфиденциальность | Не раскрыты ли внутренние данные?                                      |
| Ответственность    | Понятно ли, кто принимает финальное решение?                           |

```text
[какая ручная работа, задержка, ошибка или нагрузка есть сейчас]

[что должно стать быстрее, качественнее, дешевле или стабильнее]

[документы, таблицы, статусы, задачи, письма, обращения, источники]

1. [действие]
2. [действие]
3. [действие]

Результат:
[что должно получиться на выходе]

Проверка человеком:
[кто проверяет и что именно]

Риски:
[где ИИ может ошибиться]

Ограничения:
[какие данные нельзя использовать, какие действия нельзя выполнять автоматически]

Метрика эффекта:
[время, ошибки, ручные часы, качество, скорость, FTE, стоимость операции]

Baseline:
[как было до сценария]

Целевой эффект:
[какой результат хотим получить]

Следующий шаг:
[тест / доработка / внедрение / масштабирование / остановка]
```

## 20.12. Приложение 12. Вопросы для интервью

## Для всех ролей

1. Какие ИИ-инструменты вы реально используете в работе?
2. Какие задачи вы уже ускоряли через ИИ?
3. Как вы ставите задачу ИИ?
4. Как проверяете результат?
5. Какие данные нельзя загружать в ИИ?
6. Был ли случай, когда ИИ ошибся? Что вы сделали?
7. Есть ли у вас шаблоны, промпты или повторяемые сценарии?
8. Что вы делаете, если не знаете, как выполнить задачу?
9. Какие задачи в вашей работе можно отдать агенту?
10. Как отличаете полезное применение ИИ от имитации?

## Для руководителей

1. Какие 5 ручных задач вашей команды можно снять через ИИ?
2. Какой AI-сценарий вы запустили бы первым?
3. Как измерить эффект?
4. Где нужен human-in-the-loop?
5. Что делать с сотрудником, который говорит «я не умею»?
6. Как не превратить AI-внедрение в хаос?
7. Какую часть работы команды нельзя автоматизировать и почему?

## Для ТОПов

1. Как ИИ изменит вашу функцию за 12 месяцев?
2. Какие процессы нужно пересобрать первыми?
3. Какие роли изменятся?
4. Какие метрики покажут эффект?
5. Где главный риск?
6. Как отличить реальную AI-трансформацию от имитации?
7. Что вы будете требовать от руководителей?

```text
Затраты после:
[время / люди / ошибки / стоимость]

Эффект:
[разница]

Побочные эффекты:
[что ухудшилось / какие риски появились]

Вывод:
[масштабировать / доработать / остановить]

[действие]
```

## 20.20. Приложение 20. Мини-глоссарий

| Термин                | Краткое объяснение                                                                                    |
| --------------------- | ----------------------------------------------------------------------------------------------------- |
| **ИИ-инструмент**     | Сервис или функция, использующая искусственный интеллект для рабочих задач                            |
| **LLM**               | Языковая модель, работающая с текстом, документами, кодом и данными                                   |
| **AI-агент**          | ИИ-система, которая может выполнять задачу по шагам и использовать инструменты                        |
| **AI-сценарий**       | Повторяемый способ выполнения задачи через ИИ, агента или шаблон                                      |
| **Workflow**          | Цепочка действий в процессе                                                                           |
| **RAG**               | Подход, при котором ИИ отвечает на основе подключённых документов и базы знаний                       |
| **Human-in-the-loop** | Человек остаётся в контуре проверки и финального решения                                              |
| **Prompt**            | Инструкция или запрос к ИИ                                                                            |
| **Tasking**           | Умение ставить ИИ рабочую задачу с контекстом и критериями                                            |
| **Guardrails**        | Ограничения и проверки для безопасной работы ИИ                                                       |
| **Baseline**          | Как процесс работал до изменений                                                                      |
| **MVP**               | Быстрая проверка гипотезы без полноценной тяжёлой системы                                             |
| **Agent-first**       | Подход, где сначала проверяется, можно ли задачу выполнить через ИИ, агента, шаблон или автоматизацию |

## 20.21. Приложение 21. Короткая памятка сотрудника

## Перед задачей

1. Пойми, какой результат нужен.
2. Проверь, есть ли готовый AI-сценарий.
3. Выбери разрешённый инструмент.
4. Дай ИИ контекст, данные, ограничения и формат.
5. Получи черновик.
6. Проверь результат.
7. Доработай.
8. Используй только то, за что готов отвечать.

## Главная формула

> Не делай вручную то, что можно безопасно поручить ИИ, агенту, шаблону или автоматизированному сценарию.

## 20.22. Приложение 22. Короткая памятка руководителя

## Каждый месяц руководитель должен ответить

1. Какая ручная рутина есть в команде?
2. Что можно перевести в AI-сценарий?
3. Что уже протестировано?
4. Где есть эффект?
5. Где есть риск?
6. Кто владелец сценария?
7. Кто в команде сопротивляется?
8. Кто показывает высокий потенциал?
9. Что масштабируем?
10. Что делаем в следующем месяце?

## Главная формула

> Руководитель отвечает не только за людей, но и за способ выполнения работы: люди + ИИ + агенты + шаблоны + автоматизация + контроль качества.

> Приложения нужны для того, чтобы стандарт не остался текстом.
> Каждый раздел должен превращаться в рабочий артефакт: шаблон запроса, сценарий, чек-лист, оценку, карту рутины, тестовое задание, метрику или управленческое действие.
