# 12. Применение стандарта в адаптации сотрудников

> Рабочие материалы: [шаблон 30/60/90 адаптации (Приложение 14)](../20-appendices/20-appendices.md#2014-приложение-14-шаблон-306090-адаптации), [шаблон запроса к ИИ (Приложение 3)](../20-appendices/20-appendices.md#203-приложение-3-базовый-шаблон-запроса-к-ии), [чек-лист проверки результата (Приложение 4)](../20-appendices/20-appendices.md#204-приложение-4-чек-лист-проверки-ai-результата), [правила безопасности](../10-security/10-security.md).

Этот раздел фиксирует, как использовать стандарт AI/agent-first зрелости при адаптации новых сотрудников.

Главная идея:

> Новый сотрудник должен с первых недель привыкать к норме: рабочая задача решается не только руками, а через ИИ, цифровых агентов, шаблоны, автоматизацию и проверку результата.

---

## 12.1. Зачем стандарт нужен в адаптации

Адаптация должна не просто вводить сотрудника в должность, а сразу формировать правильную модель работы.

Стандарт помогает:

| Задача адаптации               | Как помогает стандарт                                               |
| ------------------------------ | ------------------------------------------------------------------- |
| Быстрее ввести в работу        | Сотрудник использует ИИ для изучения материалов, регламентов, задач |
| Снизить нагрузку на наставника | Часть объяснений закрывают база знаний, AI-чат, агенты              |
| Ускорить самостоятельность     | Сотрудник учится сначала пробовать через ИИ, потом эскалировать     |
| Сформировать новую норму       | С первых дней понятно: ручная рутина не является стандартом         |
| Проверить обучаемость          | Видно, как быстро сотрудник осваивает инструменты                   |
| Выявить сопротивление          | Рано видно, кто не принимает agent-first подход                     |
| Подготовить к роли             | Сотрудник выходит на минимальный AI-уровень своей позиции           |

---

## 12.2. Принцип адаптации

Адаптация должна идти не по схеме:

> Прочитал регламенты → посмотрел инструкции → спросил наставника → начал делать руками.

А по схеме:

> Получил доступ → изучил базу знаний → использовал AI-чат → применил готовый сценарий → сделал первую задачу → проверил результат → получил обратную связь.

---

## 12.3. Что новый сотрудник должен получить в первые дни

Минимальный набор:

| Элемент                                    | Что включает                                    |
| ------------------------------------------ | ----------------------------------------------- |
| **Доступ к корпоративному AI-инструменту** | AI-чат, база знаний, разрешённые агенты         |
| **Правила безопасности**                   | Что можно и нельзя загружать в ИИ               |
| **Карта инструментов**                     | Какие инструменты для каких задач использовать  |
| **Базовый шаблон запроса**                 | Как ставить задачу ИИ/агенту                    |
| **Чек-лист проверки результата**           | Как не передавать сырой AI-ответ дальше         |
| **Готовые сценарии роли**                  | Какие AI-сценарии сотрудник обязан использовать |
| **Наставник / владелец адаптации**         | Кто проверяет первые результаты                 |
| **Порог AI-зрелости роли**                 | Какой уровень A1–A6 требуется для позиции       |

---

## 12.4. Адаптация по этапам

## День 1–3. Ввод в AI/agent-first среду

Новый сотрудник должен понять:

* что компания работает в agent-first логике;
* ИИ — это базовый рабочий инструмент;
* ручная работа без попытки ускорения не считается нормой;
* результат ИИ нужно проверять;
* ответственность остаётся за человеком;
* конфиденциальные данные нельзя загружать без правил;
* незнание инструмента не является достаточным оправданием.

## Что сотрудник должен сделать

1. Получить доступ к разрешённым AI-инструментам.
2. Изучить правила безопасности.
3. Ознакомиться с базовыми понятиями.
4. Пройти вводный мини-курс.
5. Выполнить первое простое задание через ИИ.

---

## Неделя 1. Базовое применение

Цель — вывести сотрудника минимум на уровень A1–A2 в реальной рабочей задаче.

Сотрудник должен уметь:

* поставить простую задачу ИИ;
* дать контекст;
* использовать шаблон запроса;
* получить черновик результата;
* проверить результат;
* объяснить, что он сделал и как использовал ИИ.

## Пример задания

> Возьми внутренний документ/инструкцию. С помощью корпоративного AI-чата подготовь краткое резюме, выдели 5 ключевых правил, 3 риска неправильного применения и список вопросов, которые нужно уточнить у наставника.

---

## Неделя 2–4. Применение в задачах роли

Цель — сотрудник начинает использовать ИИ не отдельно, а внутри своих регулярных задач.

Сотрудник должен:

* применить готовые AI-сценарии своей роли;
* использовать ИИ для подготовки рабочих материалов;
* работать с документами, таблицами или вводными;
* проверять результаты;
* получать обратную связь;
* фиксировать, где ИИ помог, а где не справился.

## Пример задания

> Выбери одну регулярную задачу своей роли. Выполни её через ИИ или готовый AI-сценарий. Покажи: исходную задачу, запрос, результат, проверку, доработку и финальную версию.

---

## 30 дней. Первая проверка

К 30-му дню сотрудник должен показать:

| Что проверяем                     | Ожидаемый результат                       |
| --------------------------------- | ----------------------------------------- |
| Использует ли ИИ в работе         | Да, на базовых задачах                    |
| Умеет ли ставить задачу           | Да, по шаблону                            |
| Умеет ли проверять результат      | Да, по чек-листу                          |
| Соблюдает ли безопасность         | Да, понимает запреты                      |
| Пользуется ли готовыми сценариями | Да, если они есть для роли                |
| Требует ли постоянной помощи      | Нет, делает первые попытки самостоятельно |
| Есть ли сопротивление             | Нет или управляемо                        |

---

## 60 дней. Закрепление

К 60-му дню сотрудник должен выйти на минимальный уровень AI-зрелости своей роли или показать устойчивый прогресс.

| Уровень роли        | Ожидаемый результат к 60-му дню       |
| ------------------- | ------------------------------------- |
| **Специалист**      | Уверенный A2                          |
| **Ведущий эксперт** | A2→A3, первые экспертные шаблоны      |
| **Старший / лид**   | A3, первый повторяемый сценарий       |
| **Руководитель**    | A3→A4, карта рутины команды           |
| **ТОП**             | A4→A5, первые процессы для пересборки |

---

## 90 дней. Итог адаптации

К 90-му дню сотрудник должен подтвердить соответствие AI/agent-first стандарту роли.

Проверяется:

* какие задачи он выполняет через ИИ;
* какие сценарии использует;
* что перестал делать вручную;
* какие шаблоны или улучшения предложил;
* как проверяет результат;
* какие ограничения понимает;
* насколько стал самостоятельнее;
* соответствует ли минимальному уровню роли.

---

## 12.5. Минимальные требования по итогам адаптации

| Уровень роли        | Минимальный результат адаптации                                              |
| ------------------- | ---------------------------------------------------------------------------- |
| **Специалист**      | Использует готовые AI-сценарии, умеет ставить задачи ИИ, проверяет результат |
| **Ведущий эксперт** | Применяет ИИ для экспертного анализа, создаёт первые чек-листы/шаблоны       |
| **Старший / лид**   | Создаёт повторяемый AI-сценарий для своей зоны                               |
| **Руководитель**    | Формирует карту ручной рутины и запускает первый AI-сценарий команды         |
| **ТОП**             | Определяет приоритетные процессы для agent-first пересборки                  |

---

## 12.6. Роль наставника

Наставник должен не просто отвечать на вопросы нового сотрудника, а учить его работать в правильной модели.

Наставник должен спрашивать:

* что ты уже попробовал сделать через ИИ;
* какой запрос сформулировал;
* какие вводные дал;
* какой результат получил;
* что проверил;
* где возникло ограничение;
* какой следующий шаг предлагаешь.

Наставник не должен сразу давать готовый ответ, если сотрудник может сначала сделать первичную попытку через AI-инструмент.

---

## 12.7. Роль руководителя

Руководитель отвечает за то, чтобы сотрудник в адаптации не закрепился в старой ручной модели.

Руководитель должен:

1. объяснить, какой AI-уровень требуется для роли;
2. показать ключевые AI-сценарии отдела;
3. дать первые задачи с применением ИИ;
4. проверять не только результат, но и способ выполнения;
5. отслеживать сопротивление;
6. давать обратную связь;
7. фиксировать прогресс в 30/60/90 дней.

---

## 12.8. Роль HR / HRBP

HR / HRBP отвечает за то, чтобы стандарт был встроен в процесс адаптации системно.

Зона ответственности HR / HRBP:

| Зона                    | Что делать                                                 |
| ----------------------- | ---------------------------------------------------------- |
| **Программа адаптации** | Включить AI/agent-first блок в обязательную адаптацию      |
| **Материалы**           | Поддерживать вводные инструкции, чек-листы, шаблоны        |
| **Контроль сроков**     | Проверять прогресс на 30/60/90 дней                        |
| **Обратная связь**      | Собирать данные от руководителя, наставника и сотрудника   |
| **Риски**               | Выявлять сопротивление и несоответствие стандарту          |
| **Развитие**            | Назначать обучение, если сотрудник не дотягивает до уровня |

---

## 12.9. Первые обязательные задания

## Задание 1. Базовая постановка задачи ИИ

**Цель:** проверить, что сотрудник умеет формулировать рабочий запрос.

**Формат:**
Сотрудник берёт простую рабочую задачу и оформляет запрос по формуле:

> роль → задача → контекст → данные → ограничения → формат → критерии качества.

---

## Задание 2. Проверка результата

**Цель:** проверить, что сотрудник не копирует AI-ответ бездумно.

**Формат:**
Сотрудник показывает:

* исходный запрос;
* первый результат;
* что было неверно или недостаточно;
* как он доработал;
* финальный результат;
* какие ограничения остались.

---

## Задание 3. Работа с документом

**Цель:** проверить умение работать с материалами.

**Формат:**
Сотрудник обрабатывает разрешённый документ через корпоративный AI-инструмент:

* делает резюме;
* выделяет ключевые правила;
* формирует вопросы;
* отмечает риски;
* готовит краткий вывод.

---

## Задание 4. Ролевая задача

**Цель:** проверить применение ИИ в конкретной должности.

Примеры:

| Роль         | Задание                                                      |
| ------------ | ------------------------------------------------------------ |
| Специалист   | Подготовить письмо, таблицу, сводку или инструкцию           |
| Эксперт      | Проанализировать кейс и сделать чек-лист проверки            |
| Лид          | Создать шаблон для повторяемой задачи                        |
| Руководитель | Найти 3 ручные рутины команды и выбрать одну для AI-сценария |
| ТОП          | Выделить 3 процесса функции для agent-first пересборки       |

---

## 12.10. Карта адаптационной проверки

| Критерий                  | 30 дней        | 60 дней        | 90 дней                |
| ------------------------- | -------------- | -------------- | ---------------------- |
| Понимает стандарт         | Да             | Да             | Да                     |
| Использует ИИ в задачах   | Базово         | Регулярно      | Уверенно               |
| Проверяет результат       | По чек-листу   | Самостоятельно | Стабильно              |
| Соблюдает безопасность    | Знает правила  | Применяет      | Не допускает нарушений |
| Использует сценарии роли  | Начал          | Регулярно      | Предлагает улучшения   |
| Создаёт свои шаблоны      | Не обязательно | Желательно     | По уровню роли         |
| Снимает ручную рутину     | Точечно        | В своей зоне   | Системно по роли       |
| Соответствует уровню роли | Частично       | Почти          | Да / риск / нет        |

---

## 12.11. Red flags в адаптации

| Red flag                                          | Что означает                      |
| ------------------------------------------------- | --------------------------------- |
| Не использует ИИ без напоминания                  | Риск закрепления старой модели    |
| Говорит «мне проще руками»                        | Сопротивление agent-first подходу |
| Не проверяет результат                            | Риск качества                     |
| Копирует AI-ответ без понимания                   | Имитация навыка                   |
| Боится инструментов и не пробует                  | Низкая адаптивность               |
| Загружает данные без разбора                      | Риск безопасности                 |
| Постоянно спрашивает наставника до первой попытки | Низкая самостоятельность          |
| Не может объяснить, как получил результат         | Нет осознанного применения        |
| Не видит рутину в своей работе                    | Не выходит за уровень исполнителя |
| Обесценивает ИИ                                   | Культурный риск                   |

---

## 12.12. Что делать при несоответствии

Если сотрудник не выходит на минимальный уровень, действия должны быть поэтапными.

| Ситуация               | Действие                                                   |
| ---------------------- | ---------------------------------------------------------- |
| Не умеет, но старается | Дать обучение, шаблоны, практику                           |
| Ошибается в запросах   | Разобрать примеры, дать формулу tasking                    |
| Не проверяет результат | Ввести обязательный чек-лист                               |
| Боится инструментов    | Дать простые безопасные задачи                             |
| Сопротивляется         | Провести управленческую обратную связь                     |
| Не считает нужным      | Зафиксировать риск соответствия роли                       |
| Имитирует применение   | Проверить на реальных задачах                              |
| Нарушает безопасность  | Остановить, разобрать, ограничить доступ при необходимости |

---

## 12.13. Итог адаптации

По итогам адаптации руководитель должен дать оценку:

| Вопрос                                   | Ответ                     |
| ---------------------------------------- | ------------------------- |
| Сотрудник понимает agent-first стандарт? | Да / частично / нет       |
| Использует ИИ в реальных задачах?        | Да / частично / нет       |
| Проверяет результат?                     | Да / частично / нет       |
| Соблюдает безопасность?                  | Да / риск / нет           |
| Соответствует минимальному уровню роли?  | Да / риск / нет           |
| Нужен план развития?                     | Да / нет                  |
| Есть ли сопротивление?                   | Нет / умеренно / критично |
| Можно ли считать адаптацию успешной?     | Да / условно / нет        |

---

## 12.14. Главный вывод раздела

> Адаптация в agent-first среде должна не просто знакомить сотрудника с компанией.
> Она должна с первых недель формировать новую производственную норму: сначала пробуй решить задачу через ИИ, агента, шаблон или сценарий; проверяй результат; отвечай за качество; не закрепляй ручную рутину как стандарт.
