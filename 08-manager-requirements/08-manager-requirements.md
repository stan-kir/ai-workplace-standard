# 8. Что должен уметь руководитель в agent-first среде

> Рабочие материалы: [шаблон описания AI-сценария (Приложение 5)](../20-appendices/20-appendices.md#205-приложение-5-шаблон-описания-ai-сценария), [карта ручной рутины (Приложение 7)](../20-appendices/20-appendices.md#207-приложение-7-карта-ручной-рутины-отдела), [ежемесячный AI-отчёт (Приложение 11)](../20-appendices/20-appendices.md#2011-приложение-11-шаблон-для-руководителя-ежемесячный-ai-отчёт-команды), [памятка руководителя (Приложение 22)](../20-appendices/20-appendices.md#2022-приложение-22-короткая-памятка-руководителя).

Этот раздел фиксирует отдельный стандарт для всех сотрудников с управленческой ответственностью:

* ТОПов;
* руководителей функций;
* руководителей отделов;
* руководителей групп;
* лидов;
* старших сотрудников, которые координируют работу других.

Главная идея:

> Руководитель отвечает не только за то, чтобы сам использовать ИИ, а за то, чтобы команда перестала выполнять рутинную работу старым ручным способом.

---

## 8.1. Главная управленческая норма

Руководитель должен управлять не только людьми, но и системой выполнения работы:

> **Люди + ИИ + цифровые агенты + шаблоны + автоматизация + данные + контроль качества.**

В agent-first среде руководитель обязан задавать вопрос:

> **Какую часть этой работы должен сделать человек, какую — ИИ, какую — агент, какую — автоматизация, и где нужна проверка?**

---

## 8.2. За что отвечает руководитель

| Зона ответственности           | Что это означает                                                      |
| ------------------------------ | --------------------------------------------------------------------- |
| **Снятие рутины**              | Видеть повторяемые ручные задачи и переводить их в AI-сценарии        |
| **Производительность команды** | Добиваться роста скорости, качества и самостоятельности               |
| **Рабочие сценарии**           | Внедрять шаблоны, агентов, workflow и автоматизацию                   |
| **Обучение команды**           | Учить сотрудников работать через ИИ, а не только руками               |
| **Контроль качества**          | Не допускать слепого использования AI-результатов                     |
| **Безопасность**               | Следить, чтобы команда не нарушала правила данных                     |
| **Измерение эффекта**          | Фиксировать, что изменилось: скорость, качество, ошибки, трудозатраты |
| **Культура работы**            | Не принимать «я не умею» и «пусть ИТ сделает» как финальный ответ     |

---

## 8.3. Что руководитель обязан уметь

### 1. Видеть ручную рутину

Руководитель должен регулярно искать задачи, которые команда выполняет вручную, хотя их можно ускорить через ИИ, агента, шаблон или автоматизацию.

Примеры ручной рутины:

* регулярные отчёты;
* сводки статусов;
* обработка типовых обращений;
* первичный анализ данных;
* подготовка писем;
* подготовка презентаций;
* сверка таблиц;
* поиск ошибок;
* классификация информации;
* подготовка протоколов встреч;
* формирование задач по итогам обсуждений.

**Ключевой вопрос руководителя:**

> Почему это до сих пор делается руками?

---

### 2. Отличать задачу от процесса

Не всё нужно автоматизировать сразу.

Руководитель должен различать:

| Уровень                | Что делать                                               |
| ---------------------- | -------------------------------------------------------- |
| **Разовая задача**     | Ускорить через ИИ                                        |
| **Повторяемая задача** | Сделать шаблон или промпт                                |
| **Регулярный процесс** | Собрать AI-сценарий или workflow                         |
| **Массовый процесс**   | Подключать ИТ, интеграции, безопасность, данные          |
| **Критичный процесс**  | Внедрять только с контролем, тестами и human-in-the-loop |

Главная ошибка — сразу идти в большую автоматизацию, не проверив простой AI-сценарий.

---

### 3. Переводить задачи в AI-сценарии

Руководитель должен уметь описать задачу так, чтобы её можно было превратить в повторяемый сценарий.

Минимальное описание AI-сценария:

| Блок                | Вопрос                  |
| ------------------- | ----------------------- |
| **Задача**          | Что должно выполняться? |
| **Входные данные**  | Что подаём на вход?     |
| **Действия агента** | Что делает ИИ/агент?    |
| **Результат**       | Что должно получиться?  |
| **Проверка**        | Кто и как проверяет?    |
| **Риски**           | Где возможна ошибка?    |
| **Эффект**          | Что должно улучшиться?  |

Пример:

> Каждый день агент собирает открытые задачи отдела, выделяет просрочки, зависшие задачи, риски по срокам, готовит краткую сводку руководителю. Руководитель проверяет выводы и принимает управленческие решения.

---

### 4. Запускать MVP, а не ждать идеальную систему

Руководитель не должен начинать с большого ИТ-проекта, если задачу можно проверить проще.

Правильный порядок:

1. Описать проблему.
2. Найти повторяемую ручную часть.
3. Сделать простой сценарий через доступный ИИ.
4. Проверить на 5–10 примерах.
5. Оценить эффект.
6. Улучшить шаблон.
7. Только потом решать, нужна ли автоматизация или интеграция.

**Норма:**

> Сначала рабочий MVP, потом масштабирование.

---

### 5. Требовать от команды agent-first мышления

Руководитель должен перестать принимать ответы:

* «я не знаю»;
* «я не успел»;
* «нужен ИТ»;
* «так всегда делали»;
* «я не умею пользоваться ИИ»;
* «это невозможно без аналитика»;
* «надо нанять ещё человека».

Правильный управленческий вопрос:

> Что ты уже попробовал сделать через ИИ, агента, шаблон или автоматизацию?

---

### 6. Обучать команду через реальные задачи

Руководитель должен обучать команду не абстрактным курсам, а через рабочие задачи.

Формат обучения:

| Формат                   | Пример                                         |
| ------------------------ | ---------------------------------------------- |
| **Разбор задачи**        | Как эту задачу поставить агенту                |
| **Шаблон**               | Готовый промпт для регулярной работы           |
| **Совместная практика**  | Вместе прогнать задачу через ИИ                |
| **Чек-лист**             | Как проверить результат                        |
| **Библиотека сценариев** | Где лежат рабочие AI-сценарии команды          |
| **Разбор ошибок**        | Почему ИИ дал плохой результат и как исправить |

---

### 7. Контролировать качество AI-результатов

Руководитель должен следить, чтобы команда не подменяла работу сырой генерацией.

Плохой результат:

> «Вот что выдал ИИ».

Нормальный результат:

> «Я поставил задачу, получил черновик, проверил, доработал, вот ограничения и рекомендации».

Руководитель должен проверять:

* был ли понятен запрос;
* были ли корректные вводные;
* был ли результат проверен;
* есть ли ошибки;
* можно ли результат использовать;
* не нарушены ли правила безопасности;
* не ухудшилось ли качество работы.

---

### 8. Измерять эффект

ИИ-сценарии должны внедряться не ради моды, а ради измеримого результата.

Примеры метрик:

| Метрика                            | Что показывает                                  |
| ---------------------------------- | ----------------------------------------------- |
| **Время выполнения задачи**        | Стало быстрее или нет                           |
| **Количество ручных операций**     | Сколько действий сняли с человека               |
| **Количество ошибок**              | Стало качественнее или нет                      |
| **Скорость подготовки материалов** | Быстрее ли появляются отчёты, сводки, документы |
| **Повторяемость результата**       | Можно ли стабильно использовать сценарий        |
| **Загрузка команды**               | Снизилась ли рутина                             |
| **Скорость принятия решений**      | Быстрее ли руководитель получает выводы         |
| **Зависимость от ИТ/аналитиков**   | Меньше ли ожиданий внешнего ресурса             |

---

### 9. Встраивать ИИ в регулярный управленческий ритм

Руководитель должен использовать agent-first подход в регулярном управлении:

| Управленческий процесс    | Как может использоваться ИИ                                           |
| ------------------------- | --------------------------------------------------------------------- |
| **Еженедельная планёрка** | Агент готовит сводку задач, рисков и просрочек                        |
| **1:1**                   | ИИ помогает подготовить структуру разговора и follow-up               |
| **Отчётность**            | Агент собирает черновик отчёта по шаблону                             |
| **Анализ проблем**        | ИИ помогает разложить симптом, причину, гипотезы и действия           |
| **Контроль поручений**    | Агент выделяет зависшие задачи                                        |
| **Подготовка решений**    | ИИ собирает варианты, риски, trade-offs                               |
| **Обучение команды**      | ИИ помогает делать инструкции и тренажёры                             |
| **Performance review**    | ИИ помогает собрать факты, примеры, динамику, но не принимает решение |

---

## 8.4. Что руководитель должен создавать

Руководитель должен создавать не только задачи для людей, но и рабочую инфраструктуру повышения производительности.

Минимальный набор:

| Артефакт                       | Зачем нужен                                          |
| ------------------------------ | ---------------------------------------------------- |
| **Карта ручной рутины отдела** | Понять, что можно снять с людей                      |
| **Библиотека AI-сценариев**    | Повторяемые способы выполнения задач                 |
| **Шаблоны промптов**           | Быстрое и единообразное использование ИИ             |
| **Чек-листы проверки**         | Контроль качества результата                         |
| **Инструкции для команды**     | Чтобы сценарий мог использовать не один человек      |
| **Список ограничений**         | Что нельзя отдавать ИИ или нужно проверять человеком |
| **Метрики эффекта**            | Доказать, что изменения работают                     |

---

## 8.5. Минимальная карта ручной рутины отдела

Руководитель должен уметь составить простую карту ручной рутины.

| Задача                    |     Частота | Кто делает   | Сколько времени | Можно ли через ИИ/агента | Риск    | Следующий шаг        |
| ------------------------- | ----------: | ------------ | --------------: | ------------------------ | ------- | -------------------- |
| Еженедельный отчёт        |  1 раз/нед. | Менеджер     |          3 часа | Да                       | Средний | Сделать шаблон       |
| Протокол встречи          | 3 раза/нед. | Руководитель |           1 час | Да                       | Низкий  | Подключить агента    |
| Проверка таблицы          |  1 раз/день | Аналитик     |          2 часа | Частично                 | Средний | Тест на 10 файлах    |
| Ответы на типовые вопросы |   ежедневно | Специалисты  |          4 часа | Да                       | Средний | Создать базу ответов |

---

## 8.6. Минимальный шаблон AI-сценария для руководителя

Каждый руководитель должен уметь описать AI-сценарий в таком формате:

### Название сценария

Кратко: что делает сценарий.

### Проблема

Какая ручная работа, задержка, ошибка или нагрузка есть сейчас.

### Цель

Что должно стать быстрее, качественнее, дешевле или стабильнее.

### Входные данные

Какие документы, таблицы, системы, тексты или статусы нужны.

### Действия ИИ / агента

Что именно делает ИИ:

* анализирует;
* классифицирует;
* проверяет;
* резюмирует;
* сравнивает;
* формирует черновик;
* предлагает действия;
* создаёт задачи;
* готовит отчёт.

### Результат

Что должно получиться на выходе.

### Проверка человеком

Кто проверяет и что именно.

### Риски

Где ИИ может ошибиться.

### Метрика эффекта

Как понять, что сценарий полезен.

### Следующий шаг

Что делаем после проверки MVP.

---

## 8.7. Пример AI-сценария

### Название

AI-сводка по задачам отдела

### Проблема

Руководитель тратит время на ручной сбор статусов, просрочек и зависших задач.

### Цель

Сократить время подготовки к планёрке и быстрее видеть риски.

### Входные данные

Выгрузка из таск-трекера за неделю.

### Действия ИИ / агента

* группирует задачи по статусам;
* выделяет просрочки;
* находит задачи без движения;
* определяет повторяющиеся блокеры;
* готовит краткий вывод;
* предлагает вопросы для планёрки.

### Результат

Краткая управленческая сводка:

* ключевые риски;
* просрочки;
* зависшие задачи;
* вопросы к ответственным;
* предложения по действиям.

### Проверка человеком

Руководитель проверяет корректность выводов и принимает решения.

### Риски

* неверная интерпретация статуса;
* неполные данные;
* формальный вывод без понимания контекста.

### Метрика эффекта

* время подготовки к планёрке сокращено;
* риски выявляются раньше;
* меньше ручного сбора информации.

### Следующий шаг

Проверить сценарий 3 недели подряд и решить, нужен ли автоматический workflow.

## 8.8. Недопустимые модели поведения руководителя

| Поведение                              | Почему недопустимо                                      |
| -------------------------------------- | ------------------------------------------------------- |
| «Я сам пользуюсь ИИ, этого достаточно» | Руководитель отвечает за систему работы команды         |
| «Пусть ИТ автоматизирует»              | Сначала нужен MVP и понятное описание процесса          |
| «У сотрудников нет времени учиться»    | Это часть новой производительности                      |
| «У нас слишком специфичная работа»     | Сначала нужно проверить, что реально нельзя передать ИИ |
| «Мы и так справляемся»                 | Ручная работа может скрывать потерю скорости и качества |
| «ИИ ошибается, поэтому не используем»  | Ошибки управляются проверкой и human-in-the-loop        |
| «Нужны ещё люди»                       | Сначала проверяется потенциал снятия рутины             |
| «Я не знаю, как это внедрять»          | Для руководителя это зона развития, а не оправдание     |

---

## 8.9. Минимальный ежемесячный стандарт руководителя

Каждый руководитель должен не реже одного раза в месяц фиксировать:

1. какие повторяемые ручные задачи есть в команде;
2. какие из них можно перевести в AI-сценарий;
3. какой сценарий был протестирован;
4. какой результат получен;
5. сколько времени/ошибок/ручной работы удалось снять;
6. какие ограничения выявлены;
7. что будет улучшено в следующем месяце.

---

## 8.10. Главный вывод раздела

> Руководитель в agent-first среде управляет не только людьми.
> Он управляет способом выполнения работы: что делают люди, что делают ИИ и агенты, что автоматизируется, где нужна проверка и как растёт производительность команды.
